# 课程管理需求文档

## 一、模块概述

**模块名称**：课程管理  
**对应控制器**：EduCourseController、EduCategoryController、EduChapterController、EduLessonController  
**数据库表**：edu_course、edu_category、edu_chapter、edu_lesson、edu_course_student、edu_chapter_progress  

---

## 二、核心功能

### 2.1 课程分类管理

**功能描述**：管理课程的树形分类体系

**功能点**：
- 支持多级分类（parent_id自关联）
- 分类排序
- 分类启用/禁用
- 分类下课程统计

**接口**：
```
GET    /course/category/tree    # 获取分类树
GET    /course/category/list    # 获取所有分类
POST   /course/category         # 新增分类
PUT    /course/category         # 修改分类
DELETE /course/category/{id}    # 删除分类
```

**业务规则**：
- 顶级分类parent_id=0
- 有子分类的分类不可删除
- 有课程的分类不可删除
- 排序号越小越靠前

---

### 2.2 课程管理

**功能描述**：课程的创建、编辑、发布、下架全生命周期管理

#### 2.2.1 课程CRUD

**接口**：
```
GET    /course/course/page          # 分页查询课程
GET    /course/course/{id}          # 查询课程详情
POST   /course/course               # 新增课程
PUT    /course/course               # 修改课程
DELETE /course/course/{id}          # 删除课程（逻辑删除）
GET    /course/course/stats         # 课程统计数据
POST   /course/course/{id}/view     # 增加浏览次数
POST   /course/course/{id}/learn    # 增加学习人数
```

**查询条件**：
- courseName：课程名称（模糊匹配）
- categoryId：分类ID
- teacherId：讲师ID
- status：课程状态
- minPrice/maxPrice：价格区间

**课程状态流转**：
```
草稿(0) → 已发布(1) → 已下架(2)
```

#### 2.2.2 课程字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| courseName | varchar(200) | 课程名称 |
| courseCode | varchar(50) | 课程编码（唯一） |
| categoryId | bigint | 分类ID |
| teacherId | bigint | 讲师ID |
| coverImage | text | 封面图（URL或Base64） |
| description | varchar(1000) | 课程简介 |
| detail | text | 课程详情（富文本） |
| difficulty | tinyint | 难度（1初级/2中级/3高级） |
| courseHours | int | 课时数 |
| price | decimal(10,2) | 课程价格（0=免费） |
| viewCount | bigint | 浏览次数 |
| learnCount | bigint | 学习人数 |
| rating | decimal(3,1) | 平均评分（1-5星） |
| tags | varchar(500) | 课程标签（逗号分隔） |
| prerequisites | varchar(500) | 前置课程要求 |
| status | tinyint | 状态（0草稿/1已发布/2已下架） |

---

### 2.3 章节管理

**功能描述**：按课程组织章节内容

**接口**：
```
GET    /course/chapter/list/{courseId}    # 获取课程章节列表
POST   /course/chapter                     # 新增章节
PUT    /course/chapter                     # 修改章节
DELETE /course/chapter/{id}                # 删除章节
```

**业务规则**：
- 章节属于特定课程（course_id）
- 支持排序（sort字段）
- 删除课程时级联删除章节

---

### 2.4 课时管理

**功能描述**：管理具体课时的排课信息

**接口**：
```
GET    /system/lesson/page         # 分页查询课时
GET    /system/lesson/{id}         # 查询课时详情
POST   /system/lesson              # 新增课时
PUT    /system/lesson              # 修改课时
DELETE /system/lesson/{id}         # 删除课时
POST   /system/lesson/{id}/notify  # 发送上课提醒
```

**课时字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| courseId | bigint | 课程ID |
| courseName | varchar(100) | 课程名称（冗余） |
| teacherId | bigint | 授课教师ID |
| teacherName | varchar(50) | 教师姓名（冗余） |
| location | varchar(200) | 上课地点 |
| lessonDate | date | 上课日期 |
| startTime | time | 开始时间 |
| endTime | time | 结束时间 |
| status | tinyint | 状态（0取消/1正常） |
| notified | tinyint | 是否已发送提醒（0否/1是） |

**业务规则**：
- 课时关联具体课程和教师
- 支持上课地点记录（用于通勤估算）
- 上课前自动发送提醒邮件
- 课时可取消

---

### 2.5 课程学员管理

**功能描述**：管理学员报名和课程学习进度

**数据库表**：edu_course_student

**字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| courseId | bigint | 课程ID |
| studentId | bigint | 学员ID |
| enrollTime | datetime | 报名时间 |
| startTime | datetime | 开始学习时间 |
| finishTime | datetime | 完成时间 |
| progress | decimal(5,2) | 学习进度（0-100） |
| status | tinyint | 状态（1学习中/2已完成/3已过期） |
| expireTime | datetime | 课程过期时间 |

**业务规则**：
- 学员同一课程只能报名一次（唯一索引）
- 学习进度自动计算
- 课程过期后状态变更为已过期

---

### 2.6 章节学习进度

**功能描述**：记录学员对每个章节的学习进度

**数据库表**：edu_chapter_progress

**字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| courseId | bigint | 课程ID |
| chapterId | bigint | 章节ID |
| studentId | bigint | 学员ID |
| watchDuration | int | 已观看时长（秒） |
| totalDuration | int | 总时长（秒） |
| isCompleted | tinyint | 是否完成（0否/1是） |
| lastWatchTime | datetime | 最后观看时间 |

**业务规则**：
- 学员+章节唯一（uk_chapter_student）
- 进度按观看时长计算
- 观看完成标记is_completed=1
- 自动记录最后观看时间

## 四、业务流程

### 4.1 课程创建流程

```
1. 管理员/教师创建课程（草稿状态）
   ↓
2. 填写课程基本信息
   - 课程名称、编码
   - 选择分类
   - 设置讲师
   - 上传封面
   - 编写简介和详情
   ↓
3. 设置课程属性
   - 难度等级
   - 课时数
   - 价格（0=免费）
   - 标签
   - 前置要求
   ↓
4. 添加章节内容
   ↓
5. 发布课程（status=1）
   ↓
6. 学员可见并可报名
```

### 4.2 学员学习流程

```
1. 学员浏览课程列表
   ↓
2. 查看课程详情（viewCount+1）
   ↓
3. 报名课程（创建edu_course_student记录）
   ↓
4. 开始学习（startTime记录）
   ↓
5. 观看章节视频
   - 记录watchDuration
   - 更新lastWatchTime
   - 计算学习进度
   ↓
6. 完成章节学习（isCompleted=1）
   ↓
7. 完成全部章节（learnCount+1）
   ↓
8. 课程评价（1-5星）
```

### 4.3 排课提醒流程

```
1. 管理员创建课时（edu_lesson）
   ↓
2. 系统定时任务扫描即将开始的课时
   ↓
3. 查询报名该课程的学员
   ↓
4. 发送提醒邮件/站内通知
   ↓
5. 标记notified=1
```

## 七、业务规则

1. **课程编码唯一**：courseCode不可重复
2. **分类删除限制**：有子分类或课程时不可删除
3. **章节删除限制**：有学习进度记录时不可删除
4. **学员报名限制**：同一课程只能报名一次
5. **进度自动计算**：根据watchDuration/totalDuration计算
6. **浏览次数统计**：每次查看详情viewCount+1
7. **学习人数统计**：完成课程后learnCount+1
8. **课程状态控制**：下架课程学员不可继续学习
9. **排课提醒**：上课前自动发送邮件通知
10. **数据冗余**：课时表冗余课程名和教师名（提升查询性能）
