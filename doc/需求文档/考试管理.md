# 考试管理需求文档

## 一、模块概述

**模块名称**：考试管理  
**对应控制器**：ExamPaperController、ExamQuestionController、ExamRecordController、ExamSubjectController  
**数据库表**：exam_paper、exam_question、exam_paper_question、exam_record、exam_record_detail、exam_subject、exam_answer_template  

---

## 二、核心功能

### 2.1 学科管理

**功能描述**：管理考试学科分类

**接口**：
```
GET    /exam/subject/page     # 分页查询学科
GET    /exam/subject/list     # 获取所有学科
POST   /exam/subject          # 新增学科
PUT    /exam/subject          # 修改学科
DELETE /exam/subject/{id}     # 删除学科
```

**学科字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| subjectName | varchar(100) | 学科名称 |
| subjectCode | varchar(50) | 学科编码 |
| description | varchar(500) | 学科描述 |
| grade | varchar(50) | 所属年级（如"初一"、"高三"） |
| icon | varchar(50) | 学科图标（emoji） |
| color | varchar(20) | 学科颜色（十六进制） |
| sortOrder | int | 排序 |
| status | tinyint | 状态（0禁用/1启用） |

**业务规则**：
- 学科编码唯一
- 年级为字符串字段（非外键）
- 支持图标和颜色自定义

---

### 2.2 题库管理

**功能描述**：管理考试题目，支持多种题型

#### 2.2.1 题目CRUD

**接口**：
```
GET    /exam/question/page          # 分页查询题目
GET    /exam/question/{id}          # 查询题目详情
POST   /exam/question               # 新增题目
PUT    /exam/question               # 修改题目
DELETE /exam/question/{id}          # 删除题目
DELETE /exam/question/batch         # 批量删除
```

**查询条件**：
- questionTitle：题目内容（模糊匹配）
- questionType：题型
- subjectId：学科ID
- grade：年级
- difficulty：难度
- status：状态

#### 2.2.2 题目类型

| 题型编码 | 题型名称 | 说明 |
|---------|---------|------|
| 1 | 单选题 | 选择一个正确答案 |
| 2 | 多选题 | 选择多个正确答案 |
| 3 | 判断题 | 对/错 |
| 4 | 填空题 | 填写答案 |
| 5 | 简答题 | 文本回答 |

#### 2.2.3 题目字段

| 字段 | 类型 | 说明 |
|------|------|------|
| subjectId | bigint | 学科ID |
| grade | varchar(50) | 年级 |
| questionType | tinyint | 题型（1-5） |
| questionTitle | text | 题目内容 |
| options | text | 选项JSON（仅选择题） |
| correctAnswer | text | 正确答案 |
| analysis | text | 答案解析 |
| difficulty | tinyint | 难度（1简单/2中等/3困难） |
| score | decimal(5,2) | 默认分值 |
| knowledgePoint | varchar(500) | 知识点（逗号分隔） |
| usageCount | int | 使用次数 |
| status | tinyint | 状态（0禁用/1启用） |
| sourceType | tinyint | 题目来源（1手动/2导入/3解析） |

#### 2.2.4 选项JSON格式

```json
[
  {"label": "A", "content": "选项A内容"},
  {"label": "B", "content": "选项B内容"},
  {"label": "C", "content": "选项C内容"},
  {"label": "D", "content": "选项D内容"}
]
```

### 2.3 题目导入导出

#### 2.3.1 Excel导入

**接口**：`POST /exam/question/import`

**导入流程**：
1. 下载导入模板
2. 填写题目数据
3. 上传Excel文件
4. 后端解析并校验
5. 返回导入结果（成功数、失败数、错误详情）

**模板列**：
- 题目内容（必填）
- 题型（必填：1单选/2多选/3判断/4填空/5简答）
- 选项（选择题必填，格式：A.xxx B.xxx C.xxx D.xxx）
- 正确答案（必填）
- 学科ID
- 年级
- 难度（1-3）
- 分值
- 知识点
- 解析

#### 2.3.2 JSON批量导入

**接口**：`POST /exam/question/batch`

```json
[
  {
    "questionTitle": "题目内容",
    "questionType": 1,
    "options": "[{\"label\":\"A\",\"content\":\"选项A\"}]",
    "correctAnswer": "A",
    "subjectId": 1,
    "grade": "高三",
    "difficulty": 2,
    "score": 10
  }
]
```

### 2.5 试卷组装

**功能描述**：通过多种方式组卷

#### 2.5.1 按难度随机组卷

**接口**：`POST /exam/paper/assemble`

**请求参数**：
```json
{
  "paperId": 1,
  "subjectId": 1,
  "grade": "高三",
  "difficultyRatio": {
    "1": 0.3,
    "2": 0.5,
    "3": 0.2
  },
  "questionCount": {
    "1": 10,
    "2": 5,
    "3": 5,
    "4": 5,
    "5": 2
  },
  "score": 10
}
```

**业务流程**：
1. 根据难度比例计算各难度题目数量
2. 从题库随机抽取题目
3. 计算总分
4. 保存到exam_paper_question表

#### 2.5.2 手动选题组卷

**接口**：`POST /exam/paper/manual-assembly`

**请求参数**：
```json
{
  "paperId": 1,
  "questionIds": [1, 2, 3, 4, 5],
  "scores": [10, 10, 10, 10, 10]
}
```

#### 2.5.3 按知识点组卷

**接口**：`POST /exam/paper/assemble-by-kp`

**请求参数**：
```json
{
  "paperId": 1,
  "subjectId": 1,
  "grade": "高三",
  "knowledgePoints": ["函数", "数列", "概率"],
  "countPerPoint": 5,
  "score": 10
}
```

**业务规则**：
- 每个知识点抽取指定数量题目
- 适合专题训练

#### 2.5.4 预览组卷

**接口**：`POST /exam/paper/preview`

**功能**：预览组卷结果，不保存到数据库

#### 2.5.5 获取可用题目数量

**接口**：`GET /exam/paper/available-count`

**功能**：查询指定条件下可用的题目数量

### 2.6 试卷导出

**功能描述**：将试卷导出为Word或PDF格式

**接口**：
```
GET /exam/paper/export/word/{paperId}    # 导出为Word
GET /exam/paper/export/pdf/{paperId}     # 导出为PDF
GET /exam/paper/print/{paperId}          # 获取打印数据
```

**设计模式**：策略模式

```java
PaperExportStrategy (接口)
  ├── PdfExportStrategy (PDF导出)
  └── WordExportStrategy (Word导出)
  
PaperExportFactory (工厂)
  └── getStrategy(type) → 返回对应策略
```

**导出内容**：
- 试卷标题
- 考试信息（时长、总分）
- 题目列表（含选项、分值）
- 答题区域

---

### 2.7 试卷解析

**功能描述**：从Word或文本解析题目导入题库

#### 2.7.1 Word试卷解析

**接口**：`POST /exam/paper/parse/word`

**功能**：
- 上传Word格式试卷
- 自动识别题目、选项、答案
- 返回解析后的题目列表
- 可批量导入题库

#### 2.7.2 文本题卷解析

**接口**：`POST /exam/paper/parse/text`

**请求参数**：
```json
{
  "content": "1. 题目内容\nA. 选项A\nB. 选项B\n答案：A",
  "gradeId": 1,
  "subjectId": 1
}
```

### 2.8 在线考试

**功能描述**：学员在线参加考试

**服务类**：OnlineExamService

**功能点**：
- 开始考试（记录开始时间）
- 答题进度保存
- 倒计时提醒
- 自动提交（超时）
- 手动提交
- 防作弊检测

**考试流程**：
```
1. 学员选择试卷
   ↓
2. 开始考试（创建exam_record）
   ↓
3. 实时答题（前端计时）
   ↓
4. 提交试卷（或超时自动提交）
   ↓
5. 系统自动判卷（客观题）
   ↓
6. 记录考试成绩
   ↓
7. 生成答题详情（exam_record_detail）
   ↓
8. 显示成绩和解析
```

**防作弊机制**：
- 切屏次数限制
- 考试时间限制
- 浏览器信息记录
- 屏幕切换次数统计
- 警告次数统计

## 七、业务规则

1. **题目来源标记**：自动识别题目来源（手动/导入/解析）
2. **使用次数统计**：题目被组卷后usageCount+1
3. **组卷随机性**：同一条件多次组卷题目不同
4. **试卷总分计算**：组卷时自动计算totalScore
5. **客观题自动判卷**：单选、多选、判断自动判分
6. **主观题人工判卷**：填空、简答需人工批改
7. **考试超时处理**：超时自动提交试卷
8. **答题详情记录**：记录每道题的用户答案和判分结果
9. **防作弊机制**：可配置切屏次数限制、时间限制等
10. **试卷归档**：考试结束后试卷归档，不可修改
11. **错题自动收录**：考试结束后自动收录错题
12. **浏览器信息记录**：记录考试使用的浏览器信息
