# 数据库设计需求文档

## 一、数据库概述

**数据库名**：edu_training  
**字符集**：utf8mb4  
**排序规则**：utf8mb4_general_ci  
**存储引擎**：InnoDB  
**逻辑删除**：dr字段（0=正常，1=删除）  
**主键策略**：AUTO自增  

---

## 二、数据库表清单

共计**30+**张表，分为四大域：

| 域 | 表数量 | 核心表 |
|----|--------|--------|
| 课程域 | 6 | edu_category、edu_course、edu_chapter、edu_lesson、edu_course_student、edu_chapter_progress |
| 考试域 | 7 | exam_subject、exam_question、exam_paper、exam_paper_question、exam_record、exam_record_detail、exam_answer_template |
| 系统域 | 20+ | sys_user、sys_role、sys_menu、sys_config_category、sys_config_item、sys_setting、sys_activity、sys_evaluation等 |
| 关联表 | 5 | sys_user_role、sys_role_menu、sys_user_menu、sys_activity_role、sys_user_parent |

## 三、课程域表结构

### 3.1 edu_category（课程分类表）

**用途**：管理课程的树形分类体系

| 字段 | 类型 | 约束 | 默认值 | 说明 |
|------|------|------|--------|------|
| id | bigint | PK, AUTO_INCREMENT | - | 主键 |
| name | varchar(100) | NOT NULL | - | 分类名称 |
| parent_id | bigint | NOT NULL | 0 | 父分类ID（0=顶级） |
| sort | int | NOT NULL | 0 | 排序号 |
| status | tinyint | NOT NULL | 1 | 状态（0禁用/1启用） |
| dr | tinyint | NOT NULL | 0 | 逻辑删除 |
| create_time | timestamp | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| update_time | timestamp | NULL | NULL | 更新时间 |

**索引**：
```sql
PRIMARY KEY (id)
INDEX idx_parent_id (parent_id)
INDEX idx_status_dr (status, dr)
```

**业务规则**：
- 顶级分类parent_id=0
- 有子分类的分类不可删除
- 有课程的分类不可删除

## 六、索引优化策略

### 6.1 索引设计原则

1. **外键字段必建索引**：提升JOIN性能
2. **查询条件字段建索引**：WHERE、ORDER BY字段
3. **联合索引优化**：高频组合查询
4. **避免过度索引**：影响写入性能

### 6.2 联合索引示例

```sql
-- 课程查询优化
INDEX idx_category_status (category_id, status, dr)
-- 覆盖：SELECT * FROM edu_course WHERE category_id=? AND status=? AND dr=0

-- 教师课程查询
INDEX idx_teacher_status (teacher_id, status, dr)
-- 覆盖：SELECT * FROM edu_course WHERE teacher_id=? AND status=? AND dr=0

-- 题目查询优化
INDEX idx_subject_type (subject_id, question_type)
-- 覆盖：SELECT * FROM exam_question WHERE subject_id=? AND question_type=?
```

```sql
id bigint NOT NULL AUTO_INCREMENT
```

### 7.2 唯一约束

```sql
-- 用户名唯一
UNIQUE INDEX uk_username (username)

-- 课程编码唯一
UNIQUE INDEX uk_course_code (course_code)

-- 学员报名唯一
UNIQUE INDEX uk_course_student (course_id, student_id)
```

### 7.3 逻辑删除

所有业务表包含dr字段：
```sql
dr tinyint NOT NULL DEFAULT 0 COMMENT '逻辑删除：0-正常 1-删除'
```

**查询规范**：
```sql
SELECT * FROM table WHERE dr = 0
```

## 九、性能优化建议

### 9.1 查询优化

**避免SELECT ***：
```sql
-- 不推荐
SELECT * FROM edu_course WHERE id = 1

-- 推荐
SELECT id, course_name, price, status FROM edu_course WHERE id = 1
```

**使用LIMIT分页**：
```sql
SELECT * FROM edu_course WHERE dr=0 LIMIT 0, 10
```

### 9.2 写入优化

**批量插入**：
```sql
INSERT INTO edu_course (course_name, status) VALUES 
  ('课程1', 1),
  ('课程2', 1),
  ('课程3', 1);
```

### 9.3 索引优化

**避免索引失效**：
```sql
-- 索引失效：使用函数
WHERE DATE(create_time) = '2024-01-01'

-- 索引有效：范围查询
WHERE create_time >= '2024-01-01 00:00:00' 
  AND create_time < '2024-01-02 00:00:00'
```
