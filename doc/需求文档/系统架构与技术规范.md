# 系统架构与技术规范需求文档

## 一、项目概述

**项目名称**：智慧教培管理系统（edu-training）  
**项目类型**：多模块Maven项目  
**服务端口**：8080  
**核心功能**：课程管理、在线考试、题库管理、用户权限、活动公告、评价管理、系统配置

## 四、数据库设计

### 4.1 数据库概览

- **数据库名**：edu_training
- **字符集**：utf8mb4
- **逻辑删除**：dr字段（0=正常，1=删除）
- **主键策略**：AUTO自增
- **公共字段**：id, create_time, update_time, dr

### 4.2 表清单

共计**30+**张表，分为四大域：

| 域 | 表数量 | 说明 |
|----|--------|------|
| 课程域 | 6 | 分类、课程、章节、课时、学员关联、学习进度 |
| 考试域 | 7 | 学科、题库、试卷、试卷题目、考试记录、答题详情、答题卡模板 |
| 系统域 | 20+ | 用户、角色、菜单、配置、活动、评价、Excel模板、邮件等 |

### 4.3 核心表关系

```
课程域：
edu_category ← edu_course → sys_user（讲师）
     ↓              ↓
edu_chapter    edu_course_student → sys_user（学员）
     ↓
edu_chapter_progress

考试域：
exam_subject ← exam_question
     ↓              ↓
exam_paper ← exam_paper_question
     ↓
exam_record → sys_user（考生）
     ↓
exam_record_detail

系统域：
sys_user → sys_user_role → sys_role → sys_role_menu → sys_menu
   ↓
sys_activity → sys_activity_role
```

## 五、核心设计模式

### 5.1 策略模式 - 试卷导出

**应用场景**：支持多种格式导出（Word、PDF）

**实现**：
```java
// 策略接口
public interface PaperExportStrategy {
    void export(ExamPaper paper, OutputStream os);
}

// PDF导出策略
public class PdfExportStrategy implements PaperExportStrategy { ... }

// Word导出策略
public class WordExportStrategy implements PaperExportStrategy { ... }

// 策略工厂
public class PaperExportFactory {
    public PaperExportStrategy getStrategy(String type) {
        // 根据类型返回对应策略
    }
}
```

**优势**：
- 易于扩展新的导出格式
- 符合开闭原则
- 策略可复用

### 5.2 策略模式 - 邮件发送

**应用场景**：支持多种邮件发送方式

**实现**：
```java
public interface MailSendStrategy {
    void send(String to, String subject, String content);
}

public class SmtpMailStrategy implements MailSendStrategy { ... }
public class ApiMailStrategy implements MailSendStrategy { ... }
```

### 5.3 责任链模式 - 登录校验

**应用场景**：登录流程多步校验

**实现**：
```java
// 校验链
验证码校验 → 用户存在性校验 → 密码校验 → 状态校验 → 登录日志记录

// 每个环节独立，可灵活组合
```

**优势**：
- 校验逻辑解耦
- 易于添加新校验环节
- 执行顺序可配置

### 5.4 建造者模式 - 试卷组装

**应用场景**：复杂试卷组装流程

**实现**：
```java
public class PaperAssemblyBuilder {
    public PaperAssemblyBuilder setSubject(Long subjectId) { ... }
    public PaperAssemblyBuilder setDifficulty(Map<Integer, Double> ratio) { ... }
    public PaperAssemblyBuilder setQuestionCount(Map<Integer, Integer> count) { ... }
    public PaperAssemblyResult build() { ... }
}
```

**优势**：
- 分步构建复杂对象
- 代码可读性强
- 参数校验集中

### 5.5 观察者模式 - 活动发布通知

**应用场景**：活动发布后通知相关用户

**实现**：
```java
// 事件发布
eventPublisher.publishEvent(new ActivityPublishedEvent(activity));

// 事件监听器
@EventListener
public void onActivityPublished(ActivityPublishedEvent event) {
    // 发送WebSocket通知
    // 发送邮件通知
    // 记录通知日志
}
```

**优势**：
- 解耦事件发布和监听
- 易于添加新的监听器
- 支持异步处理

## 六、安全机制

### 6.1 认证与授权

**认证流程**：
```
1. 获取验证码（Redis缓存，5分钟过期）
2. 用户登录（用户名+密码+验证码）
3. BCrypt密码比对
4. 生成JWT Token（有效期2小时）
5. 生成Refresh Token（有效期7天）
6. 记录登录日志
```

**授权模型**：RBAC（基于角色的访问控制）
```
用户 ←→ 角色 ←→ 菜单
```

**数据权限**：
- 超级管理员：查看全部
- 管理员：查看下级
- 教师：查看学生
- 学生：仅查看自己

### 6.2 密码安全

- BCrypt加密（10轮）
- 随机盐值
- 不可逆加密
- 每次加密结果不同

### 6.3 Token管理

- JWT Token：2小时有效期
- Refresh Token：7天有效期
- 支持Token刷新
- 登出后Token失效

### 6.4 验证码安全

- 图形验证码：4位字符
- 邮箱验证码：6位数字
- 一次性使用
- 5分钟过期

## 八、开发规范

### 8.1 实体类规范

**继承BaseEntity**：
```java
@Data
@EqualsAndHashCode(callSuper = true)
@TableName("table_name")
public class Entity extends BaseEntity {
    @TableId(type = IdType.AUTO)
    private Long id;
    
    // 业务字段
}
```

**BaseEntity字段**：
- id：主键
- dr：逻辑删除（0正常/1删除）
- createTime：创建时间（自动填充）
- updateTime：更新时间（自动填充）

### 8.2 Service规范

**接口命名**：I + 实体名 + Service
```java
public interface IUserService extends IService<User> {
}
```

**实现命名**：实体名 + ServiceImpl
```java
@Service
public class UserServiceImpl extends ServiceImpl<UserMapper, User> 
    implements IUserService {
}
```

### 8.3 Controller规范

**统一前缀**：/module/entity
```java
@Api(tags = "用户管理")
@RestController
@RequestMapping("/system/user")
@RequiredArgsConstructor
public class UserController {
    private final IUserService userService;
}
```

### 8.4 事务管理

**注解**：

@Transactional(rollbackFor = Exception.class)
