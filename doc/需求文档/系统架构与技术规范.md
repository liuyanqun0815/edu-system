public class PdfExportStrategy implements PaperExportStrategy { ... }

### 5.2 策略模式 - 邮件发送

### 5.3 责任链模式 - 登录校验

**应用场景**：登录流程多步校验

**实现**：
```java
// 抽象校验器
public abstract class LoginValidator {
    protected LoginValidator nextValidator;
    
    public LoginValidator setNext(LoginValidator next) {
        this.nextValidator = next;
        return this;
    }
    
    public final void validate(LoginContext context) {
        doValidate(context);
        if (context.isSuccess() && nextValidator != null) {
            nextValidator.validate(context);
        }
    }
    
    protected abstract void doValidate(LoginContext context);
}

// 校验器实现
@Component public class CaptchaValidator extends LoginValidator { ... }
@Component public class UserExistsValidator extends LoginValidator { ... }
@Component public class PasswordValidator extends LoginValidator { ... }
@Component public class UserStatusValidator extends LoginValidator { ... }
```

**校验流程**：
```
验证码校验 → 用户存在性 → 密码校验 → 状态校验
```

**优势**：
- 校验逻辑解耦
- 易于添加新校验环节
- 执行顺序可配置
- 符合开闭原则

### 5.6 单例模式 - 配置缓存管理器

**应用场景**：系统配置频繁读取，避免重复查询数据库

**实现**：
```java
@Component
public class ConfigCacheManager {
    private final Map<String, String> configCache = new ConcurrentHashMap<>();
    private volatile boolean initialized = false;
    
    // 双重检查锁
    public String getConfig(String configKey) {
        if (!initialized) {
            synchronized (this) {
                if (!initialized) {
                    refreshCache();
                }
            }
        }
        return configCache.get(configKey);
    }
}
```

**优势**：
- 全局唯一实例
- 线程安全（双重检查锁）
- 避免重复查询数据库
- 配置更新时刷新缓存

## 六、安全机制

### 6.1 认证与授权

**认证流程**：
```
1. 获取验证码（Redis缓存，5分钟过期）
   GET /auth/captcha
   → 生成 LineCaptcha (160×60, 4字符)
   → 返回 Base64 图片 + captchaKey

2. 用户登录
   POST /auth/login
   → 校验验证码（key + code 匹配）
   → 查询用户（username）
   → BCrypt 密码比对
   → 用户状态检查（是否禁用）
   → 生成 UUID Token
   → 记录登录日志（IP、时间）
   → 返回 Token + 用户信息

3. 获取用户信息
   GET /auth/info
   → 根据 Token 获取当前用户
   → 返回用户详情 + 角色 + 权限菜单

4. 退出登录
   POST /auth/logout
   → 清除 Token
   → 记录登出日志
```

**授权模型**：RBAC（基于角色的访问控制）
```
用户(sys_user) ──M:N──► 角色(sys_role) ──M:N──► 菜单(sys_menu)
      │                      │
      │                      ▼
      │              活动推送(sys_activity_role)
      ▼
  用户自定义菜单(sys_user_menu)
```

**数据权限**：
- **超级管理员**：全部权限，查看所有数据
- **管理员**：管理教师和学生，查看下级数据
- **教师**：管理学生，管理课程和考试
- **学生**：参加考试、查看课程
- **家长**：查看学生学习情况

**数据权限过滤示例**（登录日志查询）：
- 超级管理员：查看全部日志
- 管理员/教师：查看自己及下级用户日志
- 学生：仅查看自己的日志

### 6.2 密码安全

- BCrypt加密（10轮）
- 随机盐值
- 不可逆加密
- 每次加密结果不同
- 密码强度校验（最少6位）

### 6.3 Token管理

- **当前实现**：UUID Token（简化方案）
- **Token存储**：Redis缓存（7天过期）
- **扩展计划**：JWT Token（有效期2小时）+ Refresh Token（7天）
- 支持Token刷新机制
- 登出后Token失效

### 6.4 验证码安全

- 图形验证码：4位字符，LineCaptcha (160×60)
- 邮箱验证码：6位数字
- 一次性使用
- 5分钟过期
- Redis缓存存储

## 八、开发规范

### 8.1 参数封装规范

**超过3个参数的方法，必须封装为Query对象**：

```java
// ❌ 错误：参数过多
public PageResult<User> pageList(Integer pageNum, Integer pageSize, 
                                  String username, Integer status, Long roleId);

// ✅ 正确：使用Query对象
public PageResult<User> pageList(UserQuery query);
```

**Query对象命名规范**：
- 继承 `BaseQuery`（包含pageNum、pageSize）
- 放在对应模块的 `query/` 包下
- 命名格式：`{Entity}Query`

### 8.2 实体类规范

**继承BaseEntity**：
```java
@Data
@EqualsAndHashCode(callSuper = true)
@TableName("table_name")
public class Entity extends BaseEntity {
    @TableId(type = IdType.AUTO)
    private Long id;
    
    // 业务字段
}
```

**BaseEntity字段**：
- id：主键
- dr：逻辑删除（0正常/1删除）
- createTime：创建时间（自动填充）
- updateTime：更新时间（自动填充）

### 8.3 Service规范

**接口命名**：I + 实体名 + Service
```java
public interface IUserService extends IService<User> {
}
```

**实现命名**：实体名 + ServiceImpl
```java
@Service
@RequiredArgsConstructor
public class UserServiceImpl extends ServiceImpl<UserMapper, User> 
    implements IUserService {
    
    @Override
    public PageResult<User> pageList(UserQuery query) {
        LambdaQueryWrapper<User> wrapper = new LambdaQueryWrapper<>();
        wrapper.like(StringUtils.isNotBlank(query.getName()), User::getName, query.getName())
               .eq(query.getStatus() != null, User::getStatus, query.getStatus())
               .orderByDesc(User::getCreateTime);
        
        Page<User> page = page(new Page<>(query.getPageNum(), query.getPageSize()), wrapper);
        return PageResult.of(page.getTotal(), page.getRecords(), 
                            query.getPageNum(), query.getPageSize());
    }
}
```

### 8.4 Controller规范

**统一前缀**：/module/entity
```java
@Api(tags = "用户管理")
@RestController
@RequestMapping("/system/user")
@RequiredArgsConstructor  // 使用构造器注入
public class UserController {
    
    private final IUserService userService;
    
    @ApiOperation("分页查询")
    @GetMapping("/page")
    public JsonResult<PageResult<User>> page(UserQuery query) {
        return JsonResult.success(userService.pageList(query));
    }
}
```

**关键点**：
- 使用 `@RequiredArgsConstructor` + `private final` 注入
- 分页查询统一使用 Query 对象
- 返回值统一使用 `JsonResult.success()`
