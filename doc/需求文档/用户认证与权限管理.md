# 用户认证与权限管理需求文档

## 一、模块概述

**模块名称**：用户认证与权限管理  
**对应控制器**：AuthController、SysUserController、SysRoleController、SysMenuController  
**数据库表**：sys_user、sys_role、sys_menu、sys_user_role、sys_role_menu、sys_user_menu、user_online_log  

---

## 二、核心功能

### 2.1 用户注册

**功能描述**：新用户通过用户名、密码、邮箱、手机号注册账号

**业务流程**：
1. 前端填写注册表单（用户名、密码、确认密码、邮箱、手机号）
2. 后端校验：
   - 用户名唯一性检查
   - 邮箱唯一性检查（如填写）
   - 手机号唯一性检查（如填写）
   - 密码强度校验
3. 密码BCrypt加密存储
4. 自动分配默认角色（student-学生）
5. 返回注册成功

**接口**：
```
POST /auth/register
请求体：
{
  "username": "zhangsan",
  "password": "123456",
  "email": "zhangsan@example.com",
  "phone": "13800138000",
  "nickname": "张三"
}
```

**业务规则**：
- 用户名必填，唯一
- 密码必填，最少6位
- 邮箱、手机号选填，但填写后必须唯一
- 注册后默认状态为启用（status=1）
- 自动分配学生角色，失败不影响注册

---

### 2.2 用户登录

**功能描述**：用户通过用户名密码登录系统，获取JWT Token

**业务流程**：
1. 前端获取验证码（图形验证码）
2. 用户输入用户名、密码、验证码
3. 后端校验：
   - 验证码正确性（Redis缓存，5分钟过期）
   - 用户名存在性
   - 密码正确性（BCrypt比对）
   - 用户状态（是否禁用）
4. 生成JWT Token和Refresh Token
5. 记录登录日志（IP、时间、UserAgent）
6. 更新用户在线状态
7. 返回Token和用户信息

**接口**：
```
GET /auth/captcha
→ 返回：{ captchaKey: "xxx", captchaImg: "data:image/png;base64,..." }

POST /auth/login
请求体：
{
  "username": "zhangsan",
  "password": "123456",
  "captchaKey": "xxx",
  "captchaCode": "abcd"
}
→ 返回：{ token, refreshToken, userId, username, nickname, avatar }
```

**安全机制**：
- 验证码一次性使用，验证后立即删除
- 密码BCrypt加密，不可逆
- JWT Token有效期2小时
- Refresh Token有效期7天
- 登录日志记录IP和设备信息

---

### 2.3 密码管理

#### 2.3.1 修改密码

**功能描述**：已登录用户修改自己的密码

**接口**：
```
POST /auth/change-password?oldPassword=xxx&newPassword=xxx
请求头：X-User-Id: 123
```

**校验规则**：
- 旧密码正确性验证
- 新密码强度校验（最少6位）
- 新密码不能与旧密码相同

#### 2.3.2 忘记密码

**功能描述**：用户通过邮箱验证码重置密码

**业务流程**：
1. 用户输入用户名或邮箱
2. 系统查找用户并验证邮箱绑定
3. 生成6位验证码，发送至邮箱
4. 验证码存储至Redis（5分钟过期）
5. 用户输入验证码和新密码
6. 验证验证码正确性
7. 更新密码（BCrypt加密）
8. 清除Redis验证码

**接口**：
```
POST /auth/forgot/send-code
{ "usernameOrEmail": "zhangsan" }

POST /auth/forgot/verify-code
{ "email": "zhangsan@example.com", "verifyCode": "123456" }

POST /auth/forgot/reset
{ "email": "zhangsan@example.com", "verifyCode": "123456", "newPassword": "new123" }
```

**安全机制**：
- 验证码5分钟过期
- 验证码一次性使用
- 邮件模板美化，包含用户昵称

---

### 2.4 Token管理

#### 2.4.1 Token刷新

**功能描述**：使用Refresh Token获取新的Access Token

**接口**：
```
POST /auth/refresh-token?refreshToken=xxx
→ 返回：{ token, refreshToken }
```

**业务规则**：
- Refresh Token有效期7天
- 刷新后返回新的Token和Refresh Token
- 旧Refresh Token失效

---

### 2.5 登录日志

**功能描述**：记录用户登录登出行为，支持按权限查询

**数据权限**：
- **超级管理员**：查看全部日志
- **管理员/教师**：查看自己及下级用户日志
- **学生**：仅查看自己的日志

**接口**：
```
GET /auth/login-logs?page=1&size=10&userId=123&username=zhang
→ 返回：分页日志列表（含用户信息、登录时间、IP、在线时长）
```

**记录内容**：
- 用户ID、用户名
- 登录时间、登出时间
- 登录IP、UserAgent
- 在线时长（分钟）

---

### 2.6 用户管理

**功能描述**：管理员对用户进行CRUD操作

**接口**：
```
GET    /system/user/page          # 分页查询用户列表
GET    /system/user/{id}          # 查询用户详情
POST   /system/user               # 新增用户
PUT    /system/user               # 修改用户
DELETE /system/user/{id}          # 删除用户
POST   /system/user/{id}/reset-pwd  # 重置密码
PUT    /system/user/{id}/status   # 启用/禁用用户
```

**功能点**：
- 用户列表支持多条件筛选（用户名、手机号、状态、角色）
- 批量导入/导出用户（Excel）
- 用户角色分配
- 用户层级管理（parent_id自关联）
- 家长关联管理

**接口**：

- 角色菜单权限分配

## 四、权限模型

### 4.1 RBAC模型

```
用户(sys_user) ──M:N──► 角色(sys_role) ──M:N──► 菜单(sys_menu)
```

**关联表**：
- sys_user_role：用户角色关联
- sys_role_menu：角色菜单关联
- sys_user_menu：用户自定义菜单

### 4.2 数据权限

**层级关系**：
```
超级管理员
  └── 管理员
        └── 教师
              └── 学生
                    └── 家长
```

**权限规则**：
- 上级可查看下级数据
- 平级不可互看
- 学生仅看自己数据

---

## 五、安全机制

### 5.1 密码安全
- BCrypt加密存储
- 加密强度10轮
- 每次加密结果不同（随机盐值）

### 5.2 验证码安全
- 图形验证码：4位字符，5分钟过期
- 邮箱验证码：6位数字，5分钟过期
- 一次性使用，验证后删除

### 5.3 Token安全
- JWT Token：有效期2小时
- Refresh Token：有效期7天
- 支持Token刷新机制
- 登出后Token失效

### 5.4 登录安全
- 登录日志记录
- IP地址记录
- 设备信息记录
- 在线状态跟踪

## 七、业务规则

1. **用户名唯一**：不允许重复注册
2. **密码强度**：最少6位字符
3. **角色分配**：注册自动分配学生角色
4. **状态控制**：禁用用户无法登录
5. **数据权限**：上级可查看下级数据
6. **日志记录**：登录登出必须记录
7. **Token管理**：支持刷新机制
8. **验证码**：5分钟过期，一次性使用
