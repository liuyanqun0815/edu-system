# 智慧教培管理系统（edu-training）架构文档

---

## 一、项目概述

**项目名称**：edu-training（智慧教培管理系统）  
**项目类型**：多模块 Maven 项目  
**核心功能**：面向教育培训机构的综合管理平台，涵盖课程管理、在线考试、题库管理、用户权限管理、活动公告、评价管理、系统配置等功能模块。  
**服务端口**：8080

---

## 二、技术栈

| 技术组件 | 版本 | 用途 |
|---------|------|------|
| Java | 1.8 | 编程语言 |
| Spring Boot | 2.3.12.RELEASE | 应用框架 |
| Spring Security | - | 安全认证框架（BCrypt 密码加密） |
| MyBatis Plus | 3.4.0 | ORM 框架（驼峰映射、自增主键） |
| Druid | 1.2.2 | 数据库连接池（初始5/最大20） |
| MySQL | 5.1.49（驱动） | 关系型数据库（5.7+） |
| Hutool | 5.5.1 | Java 工具库（验证码 CaptchaUtil 等） |
| Knife4j | 3.0.1 | API 文档（Swagger 增强，中文） |
| JWT (jjwt) | 0.11.2 | Token 认证（当前使用 UUID 简化） |
| WebSocket | - | 实时消息推送 |
| Lombok | - | 代码简化 |
| Maven | - | 项目构建与依赖管理 |

### 关键配置

spring:
  profiles:
    active: dev

    db-config:
      id-type: auto

  enable: true

# application-dev.yml
spring:
  datasource:

- CSRF 禁用

## 四、数据库设计

### 4.1 数据库概览

- **数据库名**：edu_training
- **字符集**：utf8mb4
- **逻辑删除**：dr 字段（0=正常，1=删除）
- **主键策略**：AUTO 自增
- **公共字段**：id, create_time, update_time, dr

### 4.2 数据库表清单

共计 **30+** 张表，分为四大域：

| 域 | 表数量 | 说明 |
|----|--------|------|
| 课程域 | 4 | 课程分类、课程、章节、课时 |
| 考试域 | 10 | 试卷、题目、选项、考试记录等 |
| 系统域 | 18+ | 用户、角色、菜单、配置、活动等 |

## 五、核心功能点详述

### 5.1 认证与授权

#### 认证流程

```
1. 获取验证码
   GET /auth/captcha
   → 生成 LineCaptcha (160×60, 4字符)
   → ConcurrentHashMap 缓存（5分钟过期）
   → 返回 Base64 图片 + captchaKey

2. 用户登录
   POST /auth/login
   → 校验验证码（key + code 匹配）
   → 查询用户（username）
   → BCrypt 密码比对
   → 用户状态检查（是否禁用）
   → 生成 UUID Token
   → 记录登录日志（IP、时间）
   → 返回 Token + 用户信息

3. 获取用户信息
   GET /auth/info
   → 根据 Token 获取当前用户
   → 返回用户详情 + 角色 + 权限菜单

4. 退出登录
   POST /auth/logout
   → 清除 Token
   → 记录登出日志
```

#### RBAC 权限模型

```
用户(sys_user) ──M:N──► 角色(sys_role) ──M:N──► 菜单(sys_menu)
      │                      │
      │                      ▼
      │              活动推送(sys_activity_role)
      ▼
  用户自定义菜单(sys_user_menu)
```

- **超级管理员**：全部权限，查看所有数据
- **管理员**：管理教师和学生，查看下级数据
- **教师**：管理学生，管理课程和考试
- **学生**：参加考试、查看课程
- **家长**：查看学生学习情况

#### 数据权限过滤

### 6.3 责任链模式 - 登录校验链（新增）

```java
// 抽象校验器
public abstract class LoginValidator {
    protected LoginValidator nextValidator;
    
    public LoginValidator setNext(LoginValidator next) {
        this.nextValidator = next;
        return this;
    }
    
    public final void validate(LoginContext context) {
        doValidate(context);
        if (context.isSuccess() && nextValidator != null) {
            nextValidator.validate(context);
        }
    }
    
    protected abstract void doValidate(LoginContext context);
}

// 校验器实现
@Component public class CaptchaValidator extends LoginValidator { ... }
@Component public class UserExistsValidator extends LoginValidator { ... }
@Component public class PasswordValidator extends LoginValidator { ... }
@Component public class UserStatusValidator extends LoginValidator { ... }
```

**校验流程**：
```
验证码校验 → 用户存在性 → 密码校验 → 状态校验
```

**使用示例**：
```java
LoginContext context = new LoginContext();
context.setUsername("admin");
context.setPassword("123456");

LoginValidator chain = captchaValidator
    .setNext(userExistsValidator)
    .setNext(passwordValidator)
    .setNext(userStatusValidator);

chain.validate(context);
```

### 6.6 单例模式 - 配置缓存管理器（新增）

@Component
public class ConfigCacheManager {
    private final Map<String, String> configCache = new ConcurrentHashMap<>();
    private volatile boolean initialized = false;
    
    // 双重检查锁
    public String getConfig(String configKey) {
        if (!initialized) {

                if (!initialized) {
                    refreshCache();
                }
            }
        }
        return configCache.get(configKey);
    }

### 10.3 Redis 配置与使用

#### Redis 集成说明

项目使用 Redis 作为缓存中间件,用于验证码存储、会话管理、配置缓存等场景。

**依赖配置**:
```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-redis</artifactId>
</dependency>
```

**配置文件** (`application-dev.yml`):
```yaml
spring:
  redis:
    host: localhost
    port: 6379
    password:  # 如有密码则填写
    database: 0  # 默认数据库
    lettuce:
      pool:
        max-active: 8   # 最大连接数
        max-idle: 8     # 最大空闲连接
        min-idle: 0     # 最小空闲连接
        max-wait: -1ms  # 连接超时时间
```

**核心配置类** (`RedisConfig.java`):
- 自定义 `RedisTemplate<String, Object>` Bean
- 使用 `StringRedisSerializer` 序列化 Key
- 使用 `Jackson2JsonRedisSerializer` 序列化 Value
- 设置默认过期时间

**工具类** (`RedisUtils.java`):
```java
// 基础操作
redisUtils.set(key, value, expire);      // 设置缓存
redisUtils.get(key);                      // 获取缓存
redisUtils.delete(key);                   // 删除缓存
redisUtils.hasKey(key);                   // 判断key是否存在

// 验证码存储（5分钟过期）
redisUtils.set("captcha:" + captchaKey, captchaCode, 300);

// 会话Token存储（7天过期）
redisUtils.set("token:" + token, userId, 7 * 24 * 3600);
```

**使用场景**:
1. **验证码缓存**: 登录验证码存储与校验
2. **会话管理**: 用户Token存储与验证
3. **配置缓存**: `ConfigCacheManager` 使用 Redis 缓存系统配置
4. **在线状态**: 用户在线状态跟踪

**注意事项**:
- 开发环境可使用内存模式(不配置Redis),生产环境必须配置
- 验证码场景设置合理过期时间,避免内存泄漏
- Token存储使用唯一前缀,便于批量管理

#### WebSocket 实现

**NotificationWebSocket.java** - 实时通知
```java
@Component
@ServerEndpoint("/ws/notification/{userId}")
public class NotificationWebSocket {
    
    // 存储所有WebSocket会话
    private static final Map<Long, Session> SESSION_MAP = new ConcurrentHashMap<>();
    
    @OnOpen
    public void onOpen(Session session, @PathParam("userId") Long userId) {
        SESSION_MAP.put(userId, session);
    }
    
    @OnClose
    public void onClose(@PathParam("userId") Long userId) {
        SESSION_MAP.remove(userId);
    }
    
    @OnMessage
    public void onMessage(String message, Session session) {
        // 处理客户端消息
    }
    
    // 推送消息给指定用户
    public static void sendMessage(Long userId, String message) {
        Session session = SESSION_MAP.get(userId);
        if (session != null && session.isOpen()) {
            session.getAsyncRemote().sendText(message);
        }
    }
    
    // 广播消息
    public static void broadcast(String message) {
        SESSION_MAP.values().forEach(session -> {
            if (session.isOpen()) {
                session.getAsyncRemote().sendText(message);
            }
        });
    }
}
```

**使用场景**:
- 活动公告实时推送
- 考试提醒通知
- 系统消息通知
- 在线状态实时更新

### 11.3 开发规范

#### 11.3.1 参数封装规范

**超过3个参数的方法，必须封装为Query对象**：

```java
// ❌ 错误：参数过多
public PageResult<User> pageList(Integer pageNum, Integer pageSize, 
                                  String username, Integer status, Long roleId);

// ✅ 正确：使用Query对象
public PageResult<User> pageList(UserQuery query);
```

**Query对象命名规范**：
- 继承 `BaseQuery`（包含pageNum、pageSize）
- 放在对应模块的 `query/` 包下
- 命名格式：`{Entity}Query`

#### 11.3.2 设计模式使用场景

| 场景 | 推荐模式 | 示例 |
|------|---------|------|
| 多算法切换 | 策略模式 | 邮件发送、试卷导出 |
| 复杂对象构建 | 建造者模式 | 试卷组装 |
| 校验逻辑链 | 责任链模式 | 登录校验 |
| 事件驱动通知 | 观察者模式 | 活动发布 |
| 全局唯一实例 | 单例模式 | 配置缓存 |

#### 11.3.3 Controller编写规范

```java
@RestController
@RequestMapping("/system/xxx")
@Api(tags = "xxx管理")
@RequiredArgsConstructor  // 使用构造器注入
public class XxxController {
    
    private final IXxxService xxxService;
    
    @ApiOperation("分页查询")
    @GetMapping("/page")
    public JsonResult<PageResult<Xxx>> page(XxxQuery query) {
        return JsonResult.success(xxxService.pageList(query));

}
```

**关键点**：
- 使用 `@RequiredArgsConstructor` + `private final` 注入
- 分页查询统一使用 Query 对象
- 返回值统一使用 `JsonResult.success()`

#### 11.3.4 Service编写规范

```java
@Service
@RequiredArgsConstructor
public class XxxServiceImpl extends ServiceImpl<XxxMapper, Xxx> implements IXxxService {
    
    @Override
    public PageResult<Xxx> pageList(XxxQuery query) {
        LambdaQueryWrapper<Xxx> wrapper = new LambdaQueryWrapper<>();
        wrapper.like(StringUtils.isNotBlank(query.getName()), Xxx::getName, query.getName())
               .eq(query.getStatus() != null, Xxx::getStatus, query.getStatus())
               .orderByDesc(Xxx::getCreateTime);
        
        Page<Xxx> page = page(new Page<>(query.getPageNum(), query.getPageSize()), wrapper);
        return PageResult.of(page.getTotal(), page.getRecords(), 
                            query.getPageNum(), query.getPageSize());
    }
}
```

### 11.4 常见问题排查

#### Q1：启动报错 "Bean创建失败"
**解决**：检查是否缺少 `@Component`、`@Service`、`@Controller` 等注解

#### Q2：接口返回404
**解决**：
1. 检查 `@RequestMapping` 路径是否正确
2. 检查模块是否被 edu-admin 依赖
3. 检查启动类包扫描路径

#### Q3：数据库查询慢
**解决**：
1. 检查是否缺少索引
2. 避免 N+1 查询问题
3. 使用分页查询，避免全表扫描

#### Q4：验证码不生效
**解决**：
1. 检查 Redis 是否启动
2. 检查 `RedisUtils` 是否正确配置
3. 开发环境可设置环境变量 `SKIP_CAPTCHA=true` 跳过
