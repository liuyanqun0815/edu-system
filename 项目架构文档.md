<a id="一项目概述"></a>
## 一、项目概述

**项目名称**：edu-training（智慧教培管理系统）  
**项目类型**：多模块 Maven 项目  
**核心功能**：面向教育培训机构的综合管理平台，涵盖课程管理、在线考试、题库管理、用户权限管理、活动公告、评价管理、系统配置等功能模块。  
**服务端口**：8080

## 二、技术栈

| Java | 1.8 | 编程语言 |

<a id="四数据库设计"></a>
## 四、数据库设计

<a id="41-数据库概览"></a>
### 4.1 数据库概览

- **数据库名**：edu_training
- **字符集**：utf8mb4
- **逻辑删除**：dr 字段（0=正常，1=删除）
- **主键策略**：AUTO 自增
- **公共字段**：id, create_time, update_time, dr

### 4.2 数据库表清单

共计 **30+** 张表,分为四大域:

| 域 | 表数量 | 说明 |
|----|--------|------|
| 课程域 | 5 | 课程分类、课程、章节、课时、课程学员关联 |
| 考试域 | 6 | 试卷、题库、学科、考试记录等 |
| 系统域 | 20+ | 用户、角色、菜单、配置、活动、日志、Excel模板等 |

> **已清理的冗余表**: `question`、`question_option`、`subject`、`grade`(标记废弃)、`sys_config` 为遗留冗余表,
> 对应功能已统一由 `exam_question`(JSON选项)、`exam_subject`(字符串年级)、`sys_setting` + `sys_config_category`/`sys_config_item` 承担,
> 相关Java代码已同步清理。
>
> **v2.0 优化**: 新增 `edu_course_student`(课程学员)、`sys_user_login_log`(登录日志)、`sys_operation_log`(操作日志) 等表,
> 优化索引策略,补充扩展字段,详见 `database_optimization_v2.sql`
>
> **v2.3 新增**: Excel动态模板系统 (`sys_excel_template` + `sys_excel_template_column`),支持在线配置、实时预览、动态生成

<a id="51-认证与授权"></a>
### 5.1 认证与授权

<a id="认证流程"></a>
#### 认证流程

```
1. 获取验证码
   GET /auth/captcha
   → 生成 LineCaptcha (160×60, 4字符)
   → ConcurrentHashMap 缓存（5分钟过期）
   → 返回 Base64 图片 + captchaKey

2. 用户登录
   POST /auth/login
   → 校验验证码（key + code 匹配）
   → 查询用户（username）
   → BCrypt 密码比对
   → 用户状态检查（是否禁用）
   → 生成 UUID Token
   → 记录登录日志（IP、时间）
   → 返回 Token + 用户信息

3. 获取用户信息
   GET /auth/info
   → 根据 Token 获取当前用户
   → 返回用户详情 + 角色 + 权限菜单

4. 退出登录
   POST /auth/logout
   → 清除 Token
   → 记录登出日志
```

<a id="RBAC权限模型"></a>
#### RBAC 权限模型

```
用户(sys_user) ──M:N──► 角色(sys_role) ──M:N──► 菜单(sys_menu)
      │                      │
      │                      ▼
      │              活动推送(sys_activity_role)
      ▼
  用户自定义菜单(sys_user_menu)
```

- **超级管理员**：全部权限，查看所有数据
- **管理员**：管理教师和学生，查看下级数据
- **教师**：管理学生，管理课程和考试
- **学生**：参加考试、查看课程
- **家长**：查看学生学习情况

// 抽象校验器
public abstract class LoginValidator {
    protected LoginValidator nextValidator;
    
    public LoginValidator setNext(LoginValidator next) {
        this.nextValidator = next;
        return this;
    }
    
    public final void validate(LoginContext context) {
        doValidate(context);
        if (context.isSuccess() && nextValidator != null) {
            nextValidator.validate(context);
        }
    }
    
    protected abstract void doValidate(LoginContext context);
}

// 校验器实现
@Component public class CaptchaValidator extends LoginValidator { ... }
@Component public class UserExistsValidator extends LoginValidator { ... }
@Component public class PasswordValidator extends LoginValidator { ... }
@Component public class UserStatusValidator extends LoginValidator { ... }
```

```

```

LoginContext context = new LoginContext();
context.setUsername("admin");
context.setPassword("123456");

LoginValidator chain = captchaValidator
    .setNext(userExistsValidator)
    .setNext(passwordValidator)

// 事件定义（使用Lombok @Getter简化）
@Getter
public class ActivityPublishedEvent extends ApplicationEvent {
    private final Long activityId;
    private final String activityTitle;
    private final Integer activityType;
    private final List<Long> roleIds;
    
    public ActivityPublishedEvent(Object source, Long activityId, String activityTitle, 
                                   Integer activityType, List<Long> roleIds) {

        this.activityId = activityId;
        this.activityTitle = activityTitle;
        this.activityType = activityType;
        this.roleIds = roleIds;

// 事件监听器 - WebSocket推送
@Slf4j
@Component
public class WebSocketNotificationListener {
    @Async
    @EventListener
    public void onActivityPublished(ActivityPublishedEvent event) {
        log.info("[WebSocket通知] 活动发布: {} - {}, 推送角色: {}", 
                event.getActivityId(), event.getActivityTitle(), event.getRoleIds());
        
        // 构建推送消息（使用Jackson替代fastjson）
        Map<String, Object> message = new HashMap<>();
        message.put("type", "ACTIVITY_PUBLISHED");
        message.put("activityId", event.getActivityId());
        message.put("title", event.getActivityTitle());
        message.put("publishTime", LocalDateTime.now());  // 使用当前时间
        message.put("roleIds", event.getRoleIds());
        
        ObjectMapper objectMapper = new ObjectMapper();
        String jsonMessage = objectMapper.writeValueAsString(message);
        
        // 通过反射调用WebSocket广播
        broadcastToRoles(event.getRoleIds(), jsonMessage);
    }
}

// 事件监听器 - 邮件通知
@Slf4j
@Component
@RequiredArgsConstructor

**NotificationWebSocket.java** - 实时通知
```java
@Component
@ServerEndpoint("/ws/notification/{userId}")
public class NotificationWebSocket {
    
    // 存储所有WebSocket会话
    private static final Map<Long, Session> SESSION_MAP = new ConcurrentHashMap<>();
    
    @OnOpen
    public void onOpen(Session session, @PathParam("userId") Long userId) {
        SESSION_MAP.put(userId, session);
    }
    
    @OnClose
    public void onClose(@PathParam("userId") Long userId) {
        SESSION_MAP.remove(userId);
    }
    
    @OnMessage
    public void onMessage(String message, Session session) {
        // 处理客户端消息
    }
    
    // 推送消息给指定用户
    public static void sendMessage(Long userId, String message) {
        Session session = SESSION_MAP.get(userId);
        if (session != null && session.isOpen()) {
            session.getAsyncRemote().sendText(message);
        }
    }
    
    // 广播消息
    public static void broadcast(String message) {
        SESSION_MAP.values().forEach(session -> {
            if (session.isOpen()) {
                session.getAsyncRemote().sendText(message);
            }
        });
    }
}
```

<a id="113-开发规范"></a>
### 11.3 开发规范

#### 11.3.1 参数封装规范

**超过3个参数的方法，必须封装为Query对象**：

```java
// ❌ 错误：参数过多
public PageResult<User> pageList(Integer pageNum, Integer pageSize, 
                                  String username, Integer status, Long roleId);

// ✅ 正确：使用Query对象
public PageResult<User> pageList(UserQuery query);
```

**Query对象命名规范**：
- 继承 `BaseQuery`（包含pageNum、pageSize）
- 放在对应模块的 `query/` 包下
- 命名格式：`{Entity}Query`

#### 11.3.2 设计模式使用场景

| 场景 | 推荐模式 | 示例 |
|------|---------|------|
| 多算法切换 | 策略模式 | 邮件发送、试卷导出 |
| 复杂对象构建 | 建造者模式 | 试卷组装 |
| 校验逻辑链 | 责任链模式 | 登录校验 |
| 事件驱动通知 | 观察者模式 | 活动发布 |
| 全局唯一实例 | 单例模式 | 配置缓存 |

#### 11.3.3 Controller编写规范

```java
@RestController
@RequestMapping("/system/xxx")
@Api(tags = "xxx管理")
@RequiredArgsConstructor  // 使用构造器注入
public class XxxController {
    
    private final IXxxService xxxService;
    
    @ApiOperation("分页查询")
    @GetMapping("/page")
    public JsonResult<PageResult<Xxx>> page(XxxQuery query) {
        return JsonResult.success(xxxService.pageList(query));
    }
}
```

**关键点**：
- 使用 `@RequiredArgsConstructor` + `private final` 注入
- 分页查询统一使用 Query 对象
- 返回值统一使用 `JsonResult.success()`

#### 11.3.4 Service编写规范

```java
@Service
@RequiredArgsConstructor
public class XxxServiceImpl extends ServiceImpl<XxxMapper, Xxx> implements IXxxService {
    
    @Override
    public PageResult<Xxx> pageList(XxxQuery query) {
        LambdaQueryWrapper<Xxx> wrapper = new LambdaQueryWrapper<>();
        wrapper.like(StringUtils.isNotBlank(query.getName()), Xxx::getName, query.getName())
               .eq(query.getStatus() != null, Xxx::getStatus, query.getStatus())
               .orderByDesc(Xxx::getCreateTime);
        
        Page<Xxx> page = page(new Page<>(query.getPageNum(), query.getPageSize()), wrapper);
        return PageResult.of(page.getTotal(), page.getRecords(), 
                            query.getPageNum(), query.getPageSize());
    }
}
```

### 11.4 常见问题排查

#### Q1：启动报错 "Bean创建失败"
**解决**：检查是否缺少 `@Component`、`@Service`、`@Controller` 等注解

#### Q2：接口返回404
**解决**：
1. 检查 `@RequestMapping` 路径是否正确
2. 检查模块是否被 edu-admin 依赖
3. 检查启动类包扫描路径

#### Q3：数据库查询慢
**解决**：
1. 检查是否缺少索引
2. 避免 N+1 查询问题
3. 使用分页查询，避免全表扫描

#### Q4：验证码不生效
**解决**：
1. 检查 Redis 是否启动
2. 检查 `RedisUtils` 是否正确配置
3. 开发环境可设置环境变量 `SKIP_CAPTCHA=true` 跳过

#### Axios封装 (utils/request.js)

```javascript
import axios from 'axios'

const service = axios.create({
  baseURL: '',
  timeout: 10000
})

// 请求拦截器: 注入Token
service.interceptors.request.use(config => {
  const userStore = useUserStore()
  if (userStore.token) {
    config.headers['Authorization'] = `Bearer ${userStore.token}`
  }
  return config
})

// 响应拦截器: 统一错误处理
service.interceptors.response.use(
  response => response.data,
  error => {
    if (error.response?.status === 401) {
      // Token过期,跳转登录
      userStore.logout()
      window.location.href = '/login'
    }
    return Promise.reject(error)
  }
)

export default service
```

### 12.4 状态管理 (Pinia)

#### user.js - 用户状态

```javascript
export const useUserStore = defineStore('user', {
  state: () => ({
    token: localStorage.getItem('token') || '',
    userInfo: null,
    roles: [],
    permissions: []
  }),
  actions: {
    async login(username, password, captcha) {
      const res = await request.post('/auth/login', {
        username, password, captchaKey, captchaCode
      })
      this.token = res.data.token
      this.userInfo = res.data.user
      localStorage.setItem('token', this.token)
    },
    async logout() {
      await request.post('/auth/logout')
      this.token = ''
      this.userInfo = null
      localStorage.removeItem('token')
    },
    async getInfo() {
      const res = await request.get('/auth/info')
      this.userInfo = res.data.user
      this.roles = res.data.roles
      this.permissions = res.data.permissions
    }
  }
})
```

### 12.5 组合式函数 (Composables)

#### useMessage.js - 消息提示

```javascript
export function useMessage() {
  const messageStore = useMessageStore()
  
  const toast = {
    success: (msg) => messageStore.add({ type: 'success', message: msg }),
    error: (msg) => messageStore.add({ type: 'error', message: msg }),
    warning: (msg) => messageStore.add({ type: 'warning', message: msg })
  }
  
  const confirm = ({ title, message, type = 'info' }) => {
    return new Promise((resolve) => {
      messageStore.showConfirm({ title, message, type, resolve })
    })
  }
  
  return { toast, confirm }
}
```

**使用示例**:
```javascript
const { toast, confirm } = useMessage()

toast.success('保存成功')

const ok = await confirm({
  title: '删除确认',
  message: '确定要删除吗？',
  type: 'danger'
})
```

#### usePermission.js - 权限判断

```javascript
export function usePermission() {
  const userStore = useUserStore()
  
  const hasRole = (role) => {
    return userStore.roles.includes(role)
  }
  
  const hasPermission = (permission) => {
    return userStore.permissions.includes(permission)
  }
  
  return { hasRole, hasPermission }
}
```
